# Demo Seed Matrix（分析版）

更新基准：`2026-04-08`

当前版本判断：**分析版，不是最终版。**

原因：

- `T2` 的前端 36 人 demo 已进入代码，但仍是 `Done-code-only`，还没有完成全链路录屏级再验。
- `T3` 的后端 demo snapshot 仍未完成；`backend/app/db/childcare_repository.py` 的 `_demo_snapshot()` 目前只覆盖 `c-8`、`c-11` 两个孩子。
- `lib/store.tsx` 的前端 36 人里，前 `16` 个是手工故事化样本，后 `20` 个是程序生成补位样本；这些补位样本的部分元数据来自 `Math.random()`，不适合被当作稳定首选录屏 seed。
- `lib/auth/accounts.ts` 里当前 demo 家长账号只绑定 `c-1`；`lib/agent/parent-storybook-demo-seeds.ts` 的 storybook demo seed 也只覆盖 `c-1`。因此 Parent 趋势线和 Parent 微绘本在“当前可直接跑通”的意义上都应优先写成 `c-1`。
- `lib/store.tsx` 的 demo 快照模板默认没有预置 `consultations`、`interventionCards`、`reminders`；Admin 第二展示位和 Parent 闭环展示更依赖“先实际走一遍 Teacher / Consultation 主线”或 storybook demo seed 注入。

## 2. 角色主线推荐 child 列表

### Teacher 语音主线

- **首选**：`c-1`
- **强备选**：`c-8`
- **饮食风险备选**：`c-11`
- **晨曦班备选**：`c-14`、`c-16`

判断逻辑：

- `c-1` 最适合做“Teacher 引入 -> Parent 收尾”的跨角色主线。
- `c-8` 风险张力最强，而且后端 fallback 也已经认识这个孩子。
- `c-11` 适合把 Teacher 语音入口讲成“偏食 / 营养单一”的结构化问题。
- `c-14`、`c-16` 适合第二位教师账号 `u-teacher2` 的补充录屏。

### 高风险会诊

- **首选**：`c-8`
- **第二顺位**：`c-14`
- **第三顺位**：`c-16`
- **后端 fallback 友好备选**：`c-11`

判断逻辑：

- `c-8` 同时具备年龄小、分离焦虑、午睡安抚和过敏风险，最像“会诊该出场”的样子。
- `c-14` 的连续睡眠问题和情绪波动很适合会诊解释链。
- `c-16` 有环境变化敏感和声音敏感，更像 explainability 场景。
- `c-11` 虽然是饮食问题，但如果要兼顾后端 fallback，目前比 `c-14`、`c-16` 更稳。

### Admin 第二展示位

- **首选**：`c-8`
- **并列备选**：`c-11`、`c-15`
- **扩展备选**：`c-14`、`c-16`

判断逻辑：

- `c-8` 适合展示“为什么今天最该盯这个孩子”。
- `c-11` 适合讲饮食单一和家园协同。
- `c-15` 适合讲低饮水趋势和运营看板。
- `c-14`、`c-16` 更适合答辩时补充“风险不止一种”。

### Parent 趋势线

- **当前可直接跑通首选**：`c-1`
- **分析上的更强趋势候选，但待回填 parent linkage**：`c-11`、`c-15`、`c-14`

判断逻辑：

- 当前 demo 家长账号只绑定 `c-1`，因此“录屏能直接跑”的 Parent 趋势线必须优先写 `c-1`。
- 纯从趋势故事强度看，`c-11` 的偏食、`c-15` 的饮水、`c-14` 的睡眠都更适合 trend 解释，但现在不能当作已跑通事实。

### Parent 微绘本

- **当前唯一推荐**：`c-1`

判断逻辑：

- `lib/agent/parent-storybook-demo-seeds.ts` 的三套 demo seed 都只覆盖 `c-1`。
- `app/parent/storybook/page.tsx` 也会优先给 demo parent 的 `c-1` 注入该 seed。
- 因此在当前代码事实下，不应该把其他 child 写成“已适合 storybook 录屏主线”。

## 5. 风险说明

### 仍偏 demo 的地方

- `c-17` 到 `c-36` 主要是补面宽，不是故事化强样本。
- 这些生成样本的部分基础资料来自 `Math.random()`，不适合作为稳定录屏主角。
- 后端 `_demo_snapshot()` 仍只有 `c-8`、`c-11`，而且 `institutionId`、`className`、枚举值形态与前端 demo 还没有完全对齐。
- Parent storybook demo seed 只覆盖 `c-1`，不能把别的孩子写成“同等可演示”。

### 需要人工 walkthrough 再确认的地方

- `c-8` 和 `c-11` 是否在当前 backend feed 下能稳定进入 Admin 第二展示位。
- `c-15` 在当前 build 的 Admin 排序里是否持续落入前列。
- `c-14`、`c-16` 用 `u-teacher2` 录屏时，是否比 `c-8` 更稳定地产生会诊张力。
- `c-1` 的 Parent trend 响应是否满足现场网络 / backend 状态下的稳定展示要求。
