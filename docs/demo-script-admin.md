# Admin Demo Script

## 目标

- 把高风险会诊结果转成园长能执行的决策卡
- 展示 Admin 是第二展示位，而不是独立重写的一套后台

## 主路线：Admin 决策区

### 起点页面

- `/admin`

### 操作顺序

1. 先展示首页 hero stats
2. 把镜头停在“今日重点会诊 / 高风险优先事项”
3. 打开 1 到 2 个优先级条目
4. 读 source badge
5. 如需要，再进入 `/admin/agent` 展示周报或 follow-up 入口

### 建议话术

- “会诊结束以后，园长最需要的不是再看一遍原始记录，而是先知道今天最该处理哪几件事。”
- “这里的目标不是做一个更复杂的园务后台，而是把会诊结果压缩成园长办公会式的决策卡。”
- “所以 Admin 是第二展示位，它承接前面的会诊，而不是和会诊抢主线。”

### 画面必须保留

- `RiskPriorityBoard`
- source badge
- 优先级条目
- Agent / 周报入口

### 亮点

- 从单个孩子的问题升级到机构级优先级
- 把 explainability 留在决策附近，而不是藏到后台角落
- 让三端闭环更完整

### fallback 讲法

- “如果这里出现 local fallback，说明展示层在复用现有 consultation 结果，不代表 T9C 或远端聚合已经 fully 打通。”

## Admin 段收尾

- “也就是说，教师端把问题发现出来，会诊把它解释清楚，园长端再把它组织成可执行的顺序，这样机构侧的叙事就闭环了。”
